= CI - Eks with standard configuration and Pod Identity

This project should only be executed using the CI environment and will build a new Aws EKS cluster with correctly attached roles for access from your SSO role, the CI role, or the generated EKS role.

== How do deploy?

* Make sure the `account-resources` project has been executed
* Make sure the `ci-vpc` project has been run from ci
* Make sure Checkov / Linting passes as the CI for this project depends on it
* Run this project from ci

== Details

This project uses the default `resource "aws_eks_cluster" "eks_cluster" {` terraform resources to create an EKS cluster then uses the Pod Identity addon to grant initial access to the cluster.
